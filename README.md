# llm-serving-lab

A reproducible laboratory for LLM serving, benchmark design, and performance analysis.

## Environment contract

- Python: `3.11.15`
- uv: `>=0.11.28,<0.12`
- Dependency declaration: `pyproject.toml`
- Exact resolution: committed `uv.lock`
- Local environment: `.venv/` generated by uv and never committed

## First setup

The adaptation archive intentionally does not fabricate a lockfile. Generate it once on the actual development machine, review it, and commit it:

```bash
uv --version
uv python install 3.11.15
uv lock
uv lock --check
uv sync --locked
uv run --locked python -c "import llm_serving_lab; print('package-ok')"
```

After `uv.lock` is committed, normal reproducible execution is:

```bash
uv sync --locked
uv run --locked pytest -q
```

Do not hand-maintain `requirements.txt`. If an external system requires one, generate it from the committed lock with `uv export --locked` and mark it as derived.

See `weekly/W01_PLAN.md` for the Week 1 plan, `weekly/W01_DAY1_EXECUTION.md` for the uv-specific first-day runbook, and `management/STATUS.md` for current state.
